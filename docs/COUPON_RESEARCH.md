### 쿠폰 UI 패턴

올리브영은 **검색 필터 중심**의 쿠폰 시스템입니다. 상품 카드에 쿠폰 뱃지가 직접 표시되지 않고, 사용자가 검색 결과에서 필터를 통해 쿠폰 적용 가능 상품을 찾아야 합니다. 이 방식은 "쿠폰 적용 가능 상품을 먼저 필터링 → 상세 페이지에서 쿠폰 다운로드"의 2단계 UX를 제공합니다.

| 위치      | UI 요소         | 설명                       |
| --------- | --------------- | -------------------------- |
| 검색 필터 | `쿠폰상품`      | 쿠폰 적용 가능 상품 필터링 |
| 검색 필터 | `세일상품`      | 할인 중인 상품 필터링      |
| 검색 필터 | `1+1`, `2+1`    | 증정 이벤트 상품           |
| 검색 필터 | `증정상품`      | 사은품 증정 상품           |
| 상품 상세 | `장바구니 쿠폰` | 장바구니 담기 시 적용 쿠폰 |
| 상품 상세 | `상품 쿠폰`     | 해당 상품 전용 쿠폰        |

**스크래핑 포인트**: 검색 API에서 필터 파라미터(`coupon=Y`)로 쿠폰 상품만 조회 가능. 상품 상세에서 `장바구니 쿠폰`과 `상품 쿠폰` 영역을 파싱하여 쿠폰 정보 추출.

### 쿠폰 중복 사용

- ✅ 장바구니 쿠폰 + 상품 쿠폰 **중복 사용 가능**

### 쿠폰 UI 패턴

화해는 **태그 기반 복합 혜택 표시** 시스템입니다. 하나의 상품에 여러 태그(`only화해`, `무료배송`, `증정`, `쿠폰`, `최저가`)가 동시에 표시되어, 해당 상품에서 받을 수 있는 모든 혜택을 한눈에 파악할 수 있습니다. 뷰티 리뷰 앱에서 출발한 특성상, 상품 품질 정보와 혜택 정보가 함께 제공됩니다.

| 메인 배너 | 첫구매 쿠폰            | `첫 구매 20% 쿠폰 즉시 지급` |
| 상품 상세 | 브랜드 쿠폰            | 브랜드별 할인 쿠폰           |
| 검색 필터 | `쿠폰`, `단독`         | 쿠폰/단독 상품 필터링        |

**스크래핑 포인트**: 상품 목록/상세 API 응답의 `couponDiscount: true/false` 필드로 쿠폰 적용 가능 여부 판별. UI 뱃지와 JSON 필드 값이 항상 일치하므로 JSON 파싱이 더 안정적. 첫구매 20% 쿠폰은 전 상품 적용 가능.

### 기술 정보

- JSON 데이터에서 `couponDiscount` 필드로 쿠폰 적용 가능 여부 확인
- 상품 카드에 `쿠폰`, `단독` 뱃지 직접 표시
- 검색 결과에서 필터로 쿠폰/단독 상품 필터링 가능

## 요약 비교

6개 플랫폼의 쿠폰 시스템은 크게 **3가지 패턴**으로 분류됩니다:

1. **필터 중심** (올리브영): 쿠폰 상품을 검색 필터로 찾아야 함
2. **뱃지 중심** (컬리, 화해, 에이블리): 상품 카드에 쿠폰 혜택이 바로 표시됨
3. **JSON 필드 중심** (무신사, 지그재그): API 응답에서 쿠폰 정보를 추출해야 함

스크래핑 난이도는 JSON 필드 중심이 가장 쉽고 (구조화된 데이터), 뱃지 중심이 중간 (텍스트 파싱 필요), 필터 중심이 가장 어렵습니다 (UI 상호작용 필요).
